# HRMS Demo 启动说明

## 环境要求
- JDK 17+
- Maven 3.6+
- Node.js 16+
- IDEA (IntelliJ IDEA)

## 在 IDEA 中启动后端

### 方法1：使用 Maven 插件
1. 在 IDEA 中打开 `demo-simple-login/backend` 文件夹
2. 等待 Maven 依赖下载完成
3. 找到右侧 Maven 面板
4. 展开 `hrms-demo` -> `Plugins` -> `spring-boot`
5. 双击 `spring-boot:run`

### 方法2：直接运行主类
1. 打开 `HrmsDemoApplication.java`
2. 点击类名旁边的绿色运行按钮
3. 选择 "Run 'HrmsDemoApplication'"

### 验证后端启动成功
- 控制台显示 "Started HrmsDemoApplication"
- 访问 http://localhost:8080/swagger-ui.html 查看 API 文档
- 访问 http://localhost:8080/h2-console 查看数据库（JDBC URL: jdbc:h2:mem:testdb）

## 启动前端

### 在终端中执行
```bash
cd demo-simple-login/frontend
npm install
npm run dev
```

### 访问应用
打开浏览器访问: http://localhost:3000

## 技术栈说明

### 后端技术
- **Spring Boot 3.3.13**: 核心框架
- **Spring Data JPA**: 数据持久化
- **H2 Database**: 内存数据库（开发测试用）
- **JWT**: Token 认证
- **SpringDoc**: API 文档生成
- **Lombok**: 简化代码

### 前端技术
- **Next.js 15**: React 框架
- **React 19**: UI 库
- **TypeScript**: 类型安全
- **CSS**: 原生样式

## 常见问题

### 1. 后端启动失败
- 检查 JDK 版本是否为 17+
- 检查 8080 端口是否被占用
- 清理 Maven 缓存: `mvn clean`

### 2. 前端启动失败
- 删除 node_modules 重新安装: `rm -rf node_modules && npm install`
- 检查 3000 端口是否被占用

### 3. 登录失败
- 确认后端已启动
- 检查浏览器控制台是否有跨域错误
- 确认使用正确的测试账号

## 学习建议

1. **理解 JWT 认证流程**
   - 查看 `JwtUtil.java` 了解 Token 生成和验证
   - 查看 `api.ts` 了解前端如何携带 Token

2. **学习 Spring Data JPA**
   - 查看 `UserRepository.java` 了解数据访问
   - 查看 `User.java` 了解实体映射

3. **理解前后端交互**
   - 查看 `AuthController.java` 和 `index.tsx` 了解登录流程
   - 查看 `UserController.java` 和 `users.tsx` 了解 CRUD 操作

4. **学习 Next.js 路由**
   - 理解 pages 目录下的文件如何映射到路由
   - 学习 useRouter 的使用
